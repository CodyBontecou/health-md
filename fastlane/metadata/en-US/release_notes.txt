Exports are now versioned and safer for Obsidian workflows.

New
• Markdown, Obsidian Bases, JSON, and CSV identify Health.md schema v1.
• Structured exports use canonical metric units while readable Markdown can still follow Metric or Imperial display preferences.
• Health.md writes a data dictionary so plugins, scripts, and AI assistants can understand units and roll-up rules.

Guardrails
• Roll-up summaries and file type folders remain opt-in.
• Update the Obsidian plugin before enabling roll-up summaries or file type folders.
• Existing files continue to work; re-export old ranges only when you want fully consistent units/history.

Fixed
• Sleep is assigned to the night it starts, so exporting Yesterday after waking includes last night's sleep.

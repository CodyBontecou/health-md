# Health.md Pricing Analytics Worker

Cloudflare Worker + D1 ingestion endpoint for the Health.md pricing experiment.

This is intentionally **not** a UI. Data is queried with Wrangler/D1.

## Privacy contract

The worker stores only validated, coarse pricing/activation fields:

- anonymous app-install UUID generated by Health.md (`install_id`)
- event name
- experiment/variant IDs
- app version/build/platform
- paywall context
- coarse onboarding step
- free-export counts
- export target type
- coarse metric/date buckets
- product ID (`com.codybontecou.obsidianhealth.unlock` or `com.codybontecou.obsidianhealth.unlock.family`)
- purchase/restore outcome
- coarse authorization/error categories

It rejects unknown event names/properties and does **not** store raw request IPs,
user agents, HealthKit values, metric names, health dates, vault/folder/file paths,
peer device names, or exported content.

## Setup

```bash
cd worker/pricing-analytics
npm install

# Create the D1 database once.
npm run db:create

# Copy the returned database_id into wrangler.toml.

# Apply schema locally or remotely.
npm run db:migrate:local
npm run db:migrate:remote

# Optional but recommended for deployed ingestion.
wrangler secret put INGEST_TOKEN

npm run deploy
```

Deployed production endpoint:

```text
https://health-md-pricing-analytics.costream.workers.dev
```

The app has this endpoint as its release fallback. You can override it per build
with:

```bash
PRICING_ANALYTICS_ENDPOINT_URL=https://health-md-pricing-analytics.costream.workers.dev
PRICING_ANALYTICS_INGEST_TOKEN=<same token as INGEST_TOKEN, if configured>
```

`INGEST_TOKEN` is currently unset on the Worker so the production app does not
need to embed a shared token. If an ingest token is configured later, set the same
value as a release build setting; treat it as an abuse-throttling key, not a true
secret, because mobile app bundle values are extractable.

## Wrangler queries

Recent events:

```bash
wrangler d1 execute health-md-pricing-analytics --remote --command \
"SELECT received_at,event_name,variant_id,platform,paywall_context,purchase_outcome
 FROM pricing_events
 ORDER BY received_at DESC
 LIMIT 20;"
```

Pricing funnel by variant:

```bash
wrangler d1 execute health-md-pricing-analytics --remote --command \
"SELECT
   variant_id,
   COUNT(DISTINCT CASE WHEN event_name IN ('pricing_export_preview_generated','pricing_export_succeeded') THEN install_id END) AS activated_users,
   SUM(CASE WHEN event_name='pricing_paywall_shown' THEN 1 ELSE 0 END) AS paywall_views,
   SUM(CASE WHEN event_name='pricing_purchase_started' THEN 1 ELSE 0 END) AS purchase_starts,
   SUM(CASE WHEN event_name='pricing_purchase_finished' AND purchase_outcome='succeeded' THEN 1 ELSE 0 END) AS successful_purchases
 FROM pricing_events
 GROUP BY variant_id
 ORDER BY variant_id;"
```

Windowed funnel read:

```bash
wrangler d1 execute health-md-pricing-analytics --remote --command \
"SELECT
   variant_id,
   COUNT(DISTINCT CASE WHEN event_name IN ('pricing_export_preview_generated','pricing_export_succeeded') THEN install_id END) AS activated_users,
   SUM(CASE WHEN event_name='pricing_paywall_shown' THEN 1 ELSE 0 END) AS paywall_views,
   SUM(CASE WHEN event_name='pricing_purchase_finished' AND purchase_outcome='succeeded' THEN 1 ELSE 0 END) AS successful_purchases
 FROM pricing_events
 WHERE received_at >= '2026-05-18T00:00:00Z'
   AND received_at <  '2026-06-01T00:00:00Z'
 GROUP BY variant_id;"
```

Revenue still comes from App Store Connect. Join the D1 activation/paywall counts
with ASC proceeds/refunds in the experiment runbook to compute:

```text
net revenue per activated user = (ASC proceeds - refunds) / activated_users
```
